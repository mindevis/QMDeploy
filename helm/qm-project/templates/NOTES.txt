1. Секреты не создаются чартом (кроме smtp-relay). Должны существовать в namespace {{ .Release.Namespace }}:
   - qm-mysql (MYSQL_*)
   - qm-app (DB_DSN, QMNETWORK_MYSQL_DSN, JWT_SECRET, …)
{{- if .Values.monitoring.enabled }}
   При monitoring.enabled: в qm-app добавьте GRAFANA_ADMIN_PASSWORD (или задайте monitoring.grafana.adminPasswordPlain в values только для отладки).
{{- end }}
{{- if .Values.phpmyadmin.enabled }}
   phpMyAdmin: при необходимости логин в MySQL — пользователь/пароль из Secret qm-mysql (или preloadAppCredentials в values).
{{- end }}
{{- if .Values.smtpRelay.enabled }}
   При smtpRelay.enabled: Secret smtp-relay-auth (ключ SMTP_RELAY_USERS) создаётся чартом. Строка для QMAdmin/QMNetwork:
   kubectl -n {{ .Release.Namespace }} get secret smtp-relay-auth -o jsonpath='{.data.SMTP_RELAY_USERS}' | base64 -d; echo
{{- end }}

   Первичная установка: python3 scripts/create-greenfield-secrets.py -n {{ .Release.Namespace }} — см. README QMDeploy.

2. Проверка:
   kubectl -n {{ .Release.Namespace }} get pods,ingress
   kubectl -n {{ .Release.Namespace }} rollout status deployment/qmserver

3. Обновление образов:
   helm upgrade {{ .Release.Name }} . -n {{ .Release.Namespace }} --reuse-values --set imageTag=NEW
